# Todo Gravity

무거운 일이 먼저 온다.

## 아이디어

투두 리스트의 자동 정렬. 중요한 일은 위로, 덜 중요한 일은 아래로.

문제는 "중요도"를 어떻게 아느냐. 키워드 사전? 카테고리? LLM? 사용자한테 물어보기?

전부 아니다. **타이핑 자체가 데이터다.**

## 원리

사람은 중요한 일을 입력할 때 평소와 다르게 타이핑한다.
급해서 빨라질 수도, 고민하다 느려질 수도 있다. 방향은 사람마다 다르다.

하지만 **평소와 다르다는 사실 자체는 보편적이다.**

```
importance = |현재 - 평균| / 표준편차
```

|z-score|가 높으면 중요한 일. 낮으면 루틴.

사전 없음. 질문 없음. ML 없음. 중학교 통계 하나.

## 근거

| 논문 | 핵심 발견 |
|------|----------|
| Epp et al. (CHI 2011) | 키스트로크 리듬으로 감정 상태 77-88% 분류 |
| PLOS ONE (2015) | 각성(arousal)이 타이핑 타이밍에 유의미한 영향, 방향(valence)은 아님 |
| Freihaut (2021, N=924) | 보편적 키스트로크 스트레스 마커는 없음. 개인별 베이스라인에서는 ML이 성공적으로 분류 |
| Nature Sci Reports (2019) | 스마트폰 타이핑 패턴으로 우울 감지 AUC 0.89 |

"투두 입력 시 키스트로크 |z-score|로 중요도를 추정한다"는 교차점은 기존 논문에 없다.

## 실행

**웹 데모** — `todo_gravity_demo.html`을 브라우저에서 열면 끝.

**C++**
```bash
# MSVC
cl /std:c++17 /EHsc /utf-8 todo_gravity.cpp
todo_gravity.exe

# g++
g++ -std=c++17 -O2 -o todo_gravity todo_gravity.cpp
./todo_gravity
```


## 한계

- 베이스라인 수집에 최소 5개 입력 필요 (콜드스타트)
- 키보드 vs 모바일 타이핑 패턴이 다를 수 있음
- |z-score|는 "평소와 다름"을 측정하지, "중요함"을 직접 측정하지 않음 — arousal ≈ importance라는 가정에 의존
- 피로, 환경 변화 등이 노이즈로 작용할 수 있으나, 베이스라인 축적 시 대수의 법칙에 의해 상쇄됨
